Algorithm:

  • Convert each RGB value to HSL (hue, saturation, lightness)
  • Start with an empty list of sets. While some point is not in a set, pick a random point and do a flood fill starting at that point with some adjustable tolerance parameter. Thus we end up with a partition of the image into sets of pixels close in colour (most of these sets will end up containing only a few pixels though).
  • N-merging: While we have more than N sets, find the smallest set and merge it with the neighbouring set most pixels are adjacent to (this helps to reduce the effects of noise)
  • P-merging: While we have more than P colors, find the two closest colours and do a weighted merge, taking into account the size of each set

Program is unoptimised, takes about 5-10 minutes for full sized test images.
